如何用 iPad 遠端連回家中 Windows 電腦

Created2024-08-24|Updated2024-08-25|遠端連線
|Post Views:

前言

市面上有許多遠端連線工具，如 RDP、Anydesk、TeamViewer、VNC 等，各有其優缺點。根據實際測試經驗，RDP 和 Anydesk 的延遲較低，更適合日常遠端工作的需求。Anydesk 設定簡單，且相容性更佳，特別是對於非 Windows 作業系統的使用者來說，無疑是最佳選擇。而 RDP 作為 Windows 系統內建的遠端工具，能與系統緊密整合，但在設定上可能需要處理一些網路配置的問題。

因此，本文將以 RDP 為例，介紹如何在 iPad 上遠端連線至家中的 Windows 電腦。

步驟

1. 啟用 RDP 遠端桌面

  • 如果你的作業系統是專業版 Windows，那麼 RDP 就已經內建於系統中。打開設定，點選 系統 -> 遠端桌面，啟用遠端桌面連線即可。(可以參考)
  • 如果你的作業系統不是專業版 Windows，在打開設定裡的遠端桌面可能會顯示「不支援遠端桌面」的訊息。
    那麼你可能需要去 github 下載並安裝 rdpwrap 這個開源專案 。(可以參考)
    Port 預設為 3389，存在資安漏洞上面的疑慮，一般建議盡量避免直接使用。修改方法可以參考靜宜大學提供的這篇文檔。
    2. 安裝 RDP 客戶端

基本上有兩個推薦選擇：一個是微軟自己推出的免費 RDP 客戶端，另一個是Remotix，雖然 Remotix 需支付 NT$490 的費用，但它是少數能在主機電腦的虛擬機上正常輸入中文的遠端軟體。
3. 在電腦與 iPad 安裝 VPN 軟體

除非特別申請，否則家中的網路通常不是固定 IP，因此需要依賴 VPN 來進行連線。選擇一個安全且快速的 VPN 軟體非常重要，推薦使用 Tailscale。Tailscale 容易使用，基於 WireGuard 協定，具備一定程度的安全性。
4. 設定 RDP 連線

登入 Tailscale 並連線，複製遠端電腦的 IPv4，接著打開你的 RDP 客戶端新增連線，貼上IP及Port，例如：192.168.1.100:3389，然後點選連線。大功告成。
